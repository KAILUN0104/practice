## 🎨 第一部分：色彩系統

### 1.1 主色彩系統（工業極簡風格頁面）

**主背景色**：#ECECEC
這是整個網站的主要背景色，採用淺灰色設計。這個顏色選擇是為了降低眼睛疲勞，同時提供足夠的對比度讓深色文字易於閱讀。灰色系背景也強調了工業設計的簡潔性。

**主文字色**：#121212
採用接近黑色的深灰色用作正文和標題文字。這個顏色不是純黑色（#000000），而是帶有微弱的灰度偏移，使得文字在淺灰色背景上呈現出柔和但清晰的視覺效果，避免了純黑色帶來的生硬感。

**線條色**：#A0A0A0
用於頁面分割線、邊框、網格線等設計元素。這個中灰色介於背景色和文字色之間，能夠清楚地勾勒出不同區域的邊界，但不會因為過於強烈而破壞整體的簡潔感。線條的粗細統一為 1px，營造精緻的細線美學。

### 1.2 輔助色彩系統（About 頁面 - 溫暖漸變風格）

**主渐变色**：#6366f1 到 #60a5fa
這個漸變用於 About 頁面的英雄區塊（Hero Section）背景。#6366f1 是一個帶紫色調的藍色（靛藍），#60a5fa 是一個明亮的天空藍。這個從紫到藍的漸變方向是 90 度（水平左到右），創造出一種動態和溫暖的視覺氛圍，與主頁的極簡灰色形成對比。

**背景浅渐变色**：#f8fafc 到 #e0e7ff
用於頁面整體背景和卡片背景。#f8fafc 是一個幾乎白色但帶有極弱藍色調的顏色，#e0e7ff 是一個淺紫藍色。這個 135 度的對角線漸變（從左上到右下）為整個 About 頁面提供了溫柔的背景氛圍。

**卡片漸變色**：#e0e7ff 到 #f8fafc
用於個人檔案卡片（Profile Card）的背景。這個漸變與頁面背景漸變方向相同（135 度），但顏色排列相反，使得卡片在背景中呈現出微妙的視覺分層效果。

### 1.3 色彩應用規則

淺灰背景（#ECECEC）+ 深灰文字（#121212）+ 細線色（#A0A0A0）的三色組合用於所有頁面的基礎配置，包括 index.html、classpractice.html、project.html 等主要頁面。

在需要視覺突出的區域（例如 About 頁面），則採用紫藍漸變色系來創造品牌亮點。卡片和容器的設計採用了多層漸變，從深到淺或反向變化，營造出立體感和空間層次。

陰影顏色統一採用 rgba(60, 60, 120, X%)，其中 X% 在 0.08 到 0.12 之間變化。這種深紫灰色的半透明陰影相比純黑色陰影更加柔和，與整體配色系統更加協調。

## 🔤 第二部分：字體系統

### 2.1 主要字體族

**Inter 字體族**
Inter 是一個現代的無襯線字體，特別針對屏幕顯示進行了優化。在此項目中使用的字重包括：100（極細）、300（細體）和 900（超粗）。這個字體主要用於所有正文、標題和主要內容區域。Inter 的字形設計現代且中性，適合工業設計風格的網站。字體透過 Google Fonts CDN 加載：https://fonts.googleapis.com/css2?family=Inter:wght@100;300;900

**JetBrains Mono 字體族**
這是 JetBrains 公司設計的等寬字體（Monospace Font），特別適用於代碼和技術內容顯示。在此項目中使用的字重包括 300（細體）和 400（正常）。JetBrains Mono 主要用於導航欄的菜單項、時間顯示、以及所有需要等寬對齐的技術元素。等寬字體的使用強化了項目的技術感和工業風格。字體透過 Google Fonts CDN 加載：https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@300;400

**Montserrat 字體族**（About 頁面專用）
Montserrat 是一個幾何風格的無襯線字體，用於 About 頁面的設計。在此使用的字重包括 400（正常）和 700（粗體）。Montserrat 相比 Inter 更具有現代性和個性，用於創造特定頁面的獨特氛圍。

### 2.2 字體應用規則

**導航欄字體設定**
導航欄採用 JetBrains Mono 字體，字體大小統一為 0.75rem（約 12px），字重為 300（細體），所有文字轉換為大寫（text-transform: uppercase）。這樣的設計使導航欄看起來簡潔而專業。導航欄的文字不可選中（user-select: none 隱含在 cursor: default 中），保持介面的設計感。

**主標題字體設定**
主標題（H1 標籤）使用 Inter 字體，字體大小採用響應式設計：clamp(3rem, 8vw, 10rem)。這意味著最小字體為 3rem（48px），最大字體為 10rem（160px），實際大小隨視口寬度的 8% 變化。標題使用 text-transform: uppercase 轉換為大寫，字母間距設定為 -0.05em（緊湊排列），行高設定為 0.8（緊湊行距），創造出大氣而戲劇性的視覺效果。

**主標題內的粗細變化**
在 H1 標題中，部分文字使用 .thin 類別應用 font-weight: 100（極細），顏色為 #A0A0A0（線條色），另部分文字使用 .black 類別應用 font-weight: 900（超粗），整體形成粗細對比。這種設計方法稱為「字重對比設計」，能夠在一個標題內部創造豐富的視覺層次。

**副標題和正文字體設定**
副標題（H2、H3 標籤）使用 Inter 字體，H3 字體大小為 1.4rem，加粗（font-weight: 700），顏色為 #6366f1（主漸變色）。正文段落採用 font-size: 1.08rem、line-height: 1.7（行高 170%），提供良好的可讀性。段落之間的距離設定為 margin-bottom: 18px，確保內容之間有適當的視覺呼吸空間。

**技術信息文本**
所有涉及日期、代碼、API 調用或技術數據的文本採用 JetBrains Mono 等寬字體，通常字體大小為 0.8rem 或 0.75rem，以區別於常規正文。

## 📐 第三部分：布局系統

### 3.1 固定導航欄（Header）設計

**結構設定**
導航欄採用 position: fixed 固定於視口頂部，z-index: 1000 確保其始終顯示在其他內容上方。寬度為 100%（全屏寬度），高度固定為 50px。

**網格布局**
導航欄採用 CSS Grid 布局，設定為 grid-template-columns: repeat(5, 1fr)，這意味著將導航欄平均分為五等份列。五個導航項目分別為：

第一項：HOME - 返回首頁，所有頁面都有此項
第二項：TIME - 實時數位時鐘顯示，透過 JavaScript 更新
第三項：TAIPEI / TAIWAN - 地點標示信息
第四項：FOLLOW + - 帶下拉菜單的社交媒體連結
第五項：CREATE BY LIN KAI-LUN - 製作者署名

每一列之間透過 border-right: 1px solid #A0A0A0 進行分割，最後一列沒有右邊框。導航欄頂部與頁面邊界無邊距，與頁面下方內容之間透過 border-bottom: 1px solid #A0A0A0 分割。

**導航項目樣式**
每個導航項目（.header-item）採用 padding: 15px 20px，垂直居中採用 display: flex; align-items: center。文字設定為 font-family: JetBrains Mono、font-size: 0.75rem、font-weight: 300、text-transform: uppercase。文字溢出行為設定為 white-space: nowrap，確保文字不會換行。

**下拉菜單（Follow Dropdown）**
FOLLOW + 項目具有下拉菜單功能。菜單初始狀態為 display: none，當用戶點擊該項時通過 JavaScript 添加 .active 類別，使下拉菜單變為 display: block。下拉菜單位置設定為 position: absolute; top: 50px; left: 0，確保其出現在對應導航項下方。下拉菜單具有相同的邊框和分割線設計，每個菜單項 (.follow-dropdown-item) 之間用 border-bottom: 1px solid #A0A0A0 分割。

### 3.2 主內容容器設計

**容器寬度與內邊距**
主內容容器通常採用 max-width: 900px 限制最大寬度（用於卡片型佈局），或採用 padding: 4vw 的流式設計（用於全屏型佈局）。4vw 表示內邊距為視口寬度的 4%，這樣隨著屏幕尺寸變化，內邊距會自動調整，維持比例。

**外邊距設定**
容器的上下外邊距通常設定為 margin-top: 8vh、margin-bottom: 8vh（vh 表示視口高度百分比），創造出充足的垂直空間。左右外邊距通常設定為 margin-left: auto、margin-right: auto，以實現水平居中。

**背景與邊框**
主容器的背景色為 #ffffff（純白），區別於頁面背景色。容器四周應用圓角 border-radius: 18px（卡片型佈局）或 border-radius: 14px（其他元素），使硬邊設計軟化。

**陰影設計**
主容器應用投影陰影 box-shadow: 0 8px 32px rgba(60, 60, 120, 0.12)。其中 0 為水平偏移量，8px 為垂直偏移量（向下），32px 為模糊半徑（模糊程度），rgba(60, 60, 120, 0.12) 為陰影顏色（深紫灰色，透明度 12%）。這個陰影設計相比純黑色陰影更加柔和，營造出立體感但不過分強調。

### 3.3 Flexbox 彈性布局

**容器設定**
主容器採用 display: flex、flex-wrap: wrap、gap: 40px、align-items: flex-start。flex-wrap: wrap 表示當內容超過一行時自動換行。gap: 40px 設定元素之間的間距（包括行間距和列間距）。align-items: flex-start 表示所有元素向上對齐（基於容器頂部）。

**子元素彈性設定**
在卡片佈局中，卡片元素採用 flex: 1 1 250px。這個簡寫表示 flex-grow: 1（允許增長）、flex-shrink: 1（允許縮小）、flex-basis: 250px（基礎寬度）。這意味著每張卡片最少寬度為 250px，當容器寬度足夠時會均勻增長。

### 3.4 響應式設計斷點

**主要斷點**
項目的主要響應式設計斷點為 max-width: 700px。在此斷點以下（例如平板或手機），布局會發生以下變化：

容器的 flex-direction 從預設的 row 變為 column，使元素從並排排列變為上下堆疊。
容器的內邊距從 30px 30px 調整為 30px 10px，減少左右邊距以適應較小屏幕。
子元素的 flex-basis 從特定像素值變為 100%，使元素填滿整個容器寬度。

**超小屏幕設計**
在更小的屏幕上，所有 padding 和 margin 應進一步縮小，font-size 也應進行等比例縮放。導航欄在超小屏幕上可能需要特殊處理（例如轉為漢堡菜單），但當前設計未完全實裝此功能。

## 🎯 第四部分：設計元素與細部規範

### 4.1 邊框與線條

**線條顏色與粗細**
所有邊框統一採用 1px 粗細、#A0A0A0 顏色。這個細線設計是項目的視覺特徵之一，強化了工業極簡風格。邊框用於分割不同的頁面區域、導航項目、卡片邊界等。

**邊框應用場景**
導航欄的分割線、容器邊框、卡片邊框、表單邊框等均採用統一的細線規範。某些情況下，只在特定方向應用邊框（例如 border-top、border-bottom），以創造不同的視覺效果。

**特殊邊框設計**
在 About 頁面的市場信息區塊（.market-info），採用 border-left: 1px solid #A0A0A0，只在左側應用邊框，創造出視覺上的側邊強調效果。

### 4.2 圓角設計

**卡片圓角**
主容器和卡片元素採用 border-radius: 18px 或 border-radius: 14px。18px 用於較大的主容器，14px 用於內部卡片。圓角半徑的選擇基於卡片的整體大小，確保圓角不過分圓潤也不過分尖銳。

**頭像圓角**
個人頭像採用 border-radius: 50%，將正方形或矩形圖像轉換為完美圓形。頭像尺寸固定為 100px × 100px，object-fit: cover 確保圖像填滿整個容器而不變形。

**邊界圓角處理**
所有邊界元素（如按鈕、輸入框等）採用 border-radius: 8px 或 border-radius: 12px，保持與卡片圓角的視覺協調。

### 4.3 陰影系統

**容器陰影**
主容器陰影設定為 box-shadow: 0 8px 32px rgba(60, 60, 120, 0.12)。這個陰影的特點是：

垂直偏移量 8px（向下投影），創造出元素浮起的視覺效果。
模糊半徑 32px（較大的模糊），使陰影邊界柔和而不尖銳。
使用 rgba(60, 60, 120, 0.12) 深紫灰色而非黑色，與項目的冷色調相協調。
12% 的透明度確保陰影足夠微弱，不會過度吸引注意力。

**卡片陰影**
內部卡片採用較淺的陰影 box-shadow: 0 4px 16px rgba(60, 60, 120, 0.08)，其中垂直偏移量為 4px，模糊半徑為 16px，透明度為 8%。這個陰影層級低於主容器，創造出清晰的視覺層次結構。

**頭像陰影**
頭像下方應用微小陰影 box-shadow: 0 2px 8px rgba(60, 60, 120, 0.10)，垂直偏移 2px，模糊半徑 8px，透明度 10%。這個細微的陰影使頭像看起來微微浮起。

**陰影堆疊效應**
通過設定不同層級的陰影（主容器 > 卡片 > 小元素），整個頁面呈現出多層次的深度感，加強了 3D 視覺效果。

### 4.4 動畫與過渡效果

**全局過渡設定**
在 CSS 變量中定義 --transition-smooth: 0.4s cubic-bezier(0.23, 1, 0.32, 1)。這個過渡設定應用於所有需要動畫的元素，包括導航項懸停效果、下拉菜單展開等。

**過渡時長**
0.4 秒（400 毫秒）的過渡時長足以讓用戶感受到動畫效果，但又不至於過長而顯得遲鈍。

**緩動函數**
cubic-bezier(0.23, 1, 0.32, 1) 是一個自訂的三次貝茲曲線。相比線性過渡（linear），這個曲線在動畫開始和結束時較快，中間部分較慢，創造出自然的加速和減速效果。這種緩動函數使動畫看起來更加流暢和精緻。

**導航項懸停效果**
當用戶滑鼠進入導航項時，通常應用 opacity 變化或顏色變化，過渡時間為 0.4s，使用上述緩動函數。打字動畫效果在滑鼠進入導航欄時觸發，逐個顯示每個字母。

**下拉菜單動畫**
下拉菜單的展開和收起應使用 transition 實現平滑動畫，而非瞬間出現/消失。

### 4.5 間距與對齐系統

**水平間距**
相鄰元素之間的水平間距通常設定為 gap: 40px（Flexbox）或 margin-right: 各種值。導航欄項目之間沒有額外外邊距，通過 grid-template-columns 實現均勻分配。

**垂直間距**
段落之間的垂直間距為 margin-bottom: 18px。標題與正文之間的間距為 margin-bottom: 12px（標題下方）或 margin-bottom: 0（標題上方不留間距）。

**內邊距規範**
容器的內邊距為 40px（頂部）× 30px（左右）（卡片式），或 padding: 4vw（流式）。導航項的內邊距為 15px（垂直）× 20px（水平）。

## 🏗️ 第五部分：頁面結構與類型

### 5.1 頁面類型 A：首頁（index.html）- 極簡導航型

**整體結構**
首頁採用「固定導航 + 流式內容」的結構。導航欄固定於頂部，所有正文內容在其下方流動。主體內容區採用寬度限制（通常在 1200px 左右，具體取決於內容），水平居中顯示。

**主要區塊**
導航欄區塊：5 列等寬網格，高度 50px，包含主菜單和實時時鐘。
英雄區塊（Hero Section）：展示個人簡介、網站主題和視覺亮點。
內容區塊：四大主要項目卡片導航，採用網格或彈性布局排列。
頁腳區塊：製作信息和其他元信息。

**導航功能**
首頁導航欄的每個項目都可交互：HOME 連結回首頁、TIME 顯示實時時鐘、FOLLOW + 展開社交媒體鏈接菜單、其他項目可能包含頁面內導航。

**打字動畫**
當用戶滑鼠進入導航欄時，觸發打字動畫效果。文字逐個出現，創造出動態、有趣的交互體驗。該動畫應完成一次並停止，不應重複播放。

### 5.2 頁面類型 B：About 頁面（about.html）- 英雄卡片型

**整體結構**
About 頁面採用「英雄區塊 + 浮起卡片容器」的結構。與首頁不同的是，此頁面採用了溫暖的紫藍漸變色系而非灰色系。

**英雄區塊設計**
英雄區塊（.hero）佔據頁面上半部分，背景採用 linear-gradient(90deg, #6366f1 0%, #60a5fa 100%)，從左側的靛藍漸變到右側的天空藍。英雄區塊包含頁面標題「About Us」和副標題「Discover our story, mission, and the people behind our success」。標題採用大字號（font-size: 2.8rem）和白色文字。英雄區塊的內邊距為 60px（頂部）× 20px（左右）× 40px（底部）。

**卡片容器設計**

## ⚙️ 第六部分：CSS 變量與全局設定

### 6.1 CSS 變量定義

```
:root {
    --bg-color: #ECECEC;                    /* 頁面背景色 */
    --text-color: #121212;                  /* 文字顏色 */
    --line-color: #A0A0A0;                  /* 線條顏色 */
    --transition-smooth: 0.4s cubic-bezier(0.23, 1, 0.32, 1);  /* 全局過渡效果 */
    --font-main: 'Inter', sans-serif;       /* 主字體 */
    --font-mono: 'JetBrains Mono', monospace; /* 等寬字體 */
}
```

這些 CSS 變量被定義在 :root 選擇器中，使其在整個文檔內全局可用。通過使用 var() 函數（例如 background-color: var(--bg-color)），可以輕鬆維護設計的一致性。如果需要修改整個網站的色彩或字體，只需更改這些變量的值。

### 6.2 全局樣式重置

在所有元素上應用 margin: 0、padding: 0、box-sizing: border-box 的重置樣式，確保瀏覽器預設樣式不會干擾設計。cursor: default 設定整個頁面的預設游標為箭頭（而非文本選擇游標），除非在可點擊或可輸入的元素上特別設定 cursor: pointer 或 cursor: text。

### 6.3 Body 與主容器樣式

Body 元素設定 background-color: var(--bg-color)、color: var(--text-color)、font-family: var(--font-main)，確保所有文字內容採用統一的字體和顏色。padding-top: 60px 為固定導航欄留出空間（導航欄高度 50px + 額外 10px 間距）。overflow-x: hidden 防止頁面出現水平捲軸。

## 🎭 第七部分：交互設計與行為

### 7.1 導航欄交互

**Home 項目**
HOME 項目作為所有頁面的共同導航元素，點擊時應返回首頁（index.html）。無論用戶在哪個頁面，HOME 項都應始終可用。

**Time 項目**
TIME 項目透過 JavaScript 實時更新，顯示當前時間。時間格式通常為 HH:MM:SS 或 HH:MM（取決於設計決定）。時間應每秒更新一次，保持準確性。

**Follow 下拉菜單**
FOLLOW + 項目在點擊時展開下拉菜單，顯示社交媒體連結（例如 Instagram）。菜單項包含連結文字和對應的 URL。當用戶點擊菜單項時，應在新標籤頁打開對應的社交媒體頁面（target="_blank"）。點擊頁面其他位置時，下拉菜單應自動收起。

**打字動畫**
當用戶滑鼠進入導航欄時，觸發打字動畫效果。動畫應逐個顯示導航項文字的字母，創造出逐字顯示的視覺效果。動畫速度應適中（例如每個字母顯示 100-200ms），既能展示效果又不會過於冗長。動畫應完成一次後停止，除非用戶再次滑鼠進入。

### 7.2 超連結與按鈕交互

**懸停效果**
所有可點擊的超連結和按鈕在滑鼠懸停時應提供視覺反饋。反饋方式可能包括：顏色變化、背景色變化、文字下劃線、陰影加深等。過渡應使用 --transition-smooth（0.4s cubic-bezier）實現平滑效果，而非瞬間變化。

**活躍狀態**
在點擊超連結或按鈕時，應提供短暫的視覺反饋（例如顏色加深、向下凹陷效果等），表示用戶操作已被註冊。

**焦點狀態**
使用鍵盤導航時（Tab 鍵），可點擊元素應顯示明確的焦點狀態（例如外邊框或背景色變化），便於鍵盤用戶識別。

### 7.3 表單交互（如適用）

**輸入欄焦點**
當用戶點擊或使用 Tab 鍵進入輸入欄時，應顯示焦點狀態，例如邊框顏色變化、陰影變化等。邊框應採用統一的 #A0A0A0 顏色，焦點時可變為 #6366f1 或其他強調色。

**錯誤狀態**
如果表單驗證發現錯誤，應在輸入欄旁邊顯示錯誤提示文字，邊框顏色應變為紅色或其他警告色。

**提交按鈕**
提交按鈕應採用明確的視覺設計，區別於其他按鈕。懸停時應提供視覺反饋。提交過程中應顯示加載狀態（例如加載動畫或文字變化），防止用戶重複提交。

### 7.4 Vue.js 交互

**狀態切換**
在課程練習頁面或其他內容中，使用 Vue.js 實現狀態切換。例如，點擊標籤頁選項卡切換課程內容時，Vue.js 應動態更新 DOM 並應用相應的樣式。狀態變化應配合過渡動畫，使用 CSS 或 Vue.js 過渡組件實現平滑效果。

**動態內容渲染**
Vue.js 應根據數據狀態動態渲染頁面內容。例如，根據用戶選擇的過濾器條件，動態過濾和顯示產品列表。過渡動畫應使用 0.4s cubic-bezier 過渡效果。

## 📱 第八部分：響應式設計指南

### 8.1 主要設計原則

**移動優先設計**
雖然項目的起點是桌面設計，但應遵循「移動優先」的原則，確保在小屏幕上的用戶體驗不會因為版面調整而顯著下降。所有設計應考慮到平板和手機用戶。

**流式與固定混合**
某些元素（如導航欄）應保持固定尺寸，而其他元素（如內容區域）應採用流式響應式設計。使用 vw、vh、em 等相對單位使設計更加靈活。

### 8.2 關鍵斷點與調整

**最小斷點（超小屏幕 - 320px 至 480px）**
所有元素堆疊為單列。導航欄可能需要轉為漢堡菜單式設計。字體大小進一步縮小，確保可讀性。內邊距和外邊距進一步減少，最大化屏幕利用率。圖片寬度設定為 100% 以適應屏幕。

**小屏幕（480px 至 700px）**
大部分元素仍為單列排列。導航欄仍保持 5 列網格（但文字可能更緊湊）。內容容器寬度接近 100%，內邊距適度縮小。

**主要斷點（700px）**
在此斷點處，布局從單列變為多列。容器採用 Flexbox 行排列（flex-direction: row）。卡片元素可並排顯示。內邊距和外邊距恢復到桌面級別。

**中等屏幕（700px 至 1024px）**
2 至 3 列布局，取決於容器寬度。導航欄仍保持 5 列網格。字體大小適度增大。

**大屏幕（1024px 及以上）**
完整的多列布局。導航欄 5 列網格保持不變。字體大小達到最大值（如果使用 clamp 函數）。容器可能達到 max-width 上限。

### 8.3 特定元素的響應式調整

**英雄標題（H1）**
使用 clamp(3rem, 8vw, 10rem) 實現流式響應。在最小尺寸（320px）時，標題將為 3rem；在最大尺寸時為 10rem；中間過程中按視口寬度的 8% 變化。

**導航欄**
導航欄寬度始終為 100%，高度固定 50px。在超小屏幕上，文字可能溢出；此時應考慮縮小字體（例如從 0.75rem 變為 0.65rem）或改用縮寫文本（例如「HOME」可縮寫為「H」）。

**容器寬度**
流式設計時，使用 padding: 4vw 讓內邊距隨視口寬度變化。卡片式設計時，max-width: 900px 限制寬度。在小屏幕上，可設定 max-width: 100% 讓容器填滿屏幕寬度。

**網格列數**
產品網格在大屏幕上為 4 列，中等屏幕上為 3 列，小屏幕上為 2 列，超小屏幕上為 1 列。可使用 auto-fit 或 auto-fill 等 CSS Grid 技巧實現自動調整。

### 8.4 媒體查詢實踐

主要媒體查詢應設定於 max-width: 700px（及其他必要斷點）。每個媒體查詢內應包含受影響元素的所有樣式調整，避免分散到文檔各處。

## 🔧 第九部分：技術實現細節

### 9.1 HTML 結構規範

**語義化 HTML**
所有頁面應使用語義化 HTML5 元素，例如 header、nav、main、section、article、footer 等，提高代碼可讀性和無障礙性。

**導航欄標記**
導航欄應使用 header 元素包裝，每個導航項使用 div.header-item 或類似結構。FOLLOW + 下拉菜單應使用 ul/li 或 nav 元素，確保結構清晰。

**內容標記**
頁面內容應使用適當的標題標籤（h1、h2、h3 等）進行層次劃分。段落使用 p 標籤，列表使用 ul/ol 和 li 標籤。

**圖像標記**
所有圖像應使用 img 標籤，並附帶 alt 屬性以供無障礙訪問和 SEO。img 標籤應設定寬度和高度屬性以預留空間，防止版面抖動。

### 9.2 CSS 組織結構

**層級順序**
CSS 應按以下順序組織：1) CSS 變量定義 2) 全局重置和基礎樣式 3) 排版樣式（字體、字號、行高等）4) 佈局樣式（flexbox、grid、定位等）5) 組件樣式（按組件分組）6) 狀態樣式（hover、active、focus 等）7) 響應式設計（媒體查詢）。

**類別命名**
採用 BEM（Block Element Modifier）命名約定或其他一致的命名規則，例如 .header-item、.profile-card、.profile-card__image 等，提高代碼可維護性。

**避免過度特異性**
避免使用過高優先級的選擇器（如 ID 選擇器或多層選擇器）。使用類別選擇器作為主要方式。如果出現樣式衝突，應考慮重構代碼而非增加 !important。

### 9.3 JavaScript 交互實現

**事件監聽**
所有交互功能應使用事件監聽器實現，例如 addEventListener('click', handler)、addEventListener('mouseenter', handler) 等。事件處理函數應簡潔高效，避免在事件處理中執行複雜計算。

**打字動畫實現**
打字動畫可使用迴圈和延遲定時器實現，逐個顯示字符。每個字符的顯示延遲應為 100-200ms。動畫完成後應停止定時器，防止多次執行。

**時鐘更新**
時間顯示應每 1000ms（1 秒）更新一次，使用 setInterval 或 requestAnimationFrame 實現。時間格式化應考慮時區和本地化設置。

**Vue.js 集成**
如使用 Vue.js 3，應定義反應式數據屬性、計算屬性和方法。使用 v-if/v-show 控制元素顯示隱藏，使用 v-for 進行列表渲染，使用 @click/@mousenter 等指令綁定事件。

### 9.4 性能優化

**代碼分割**
對於大型項目，應將 CSS 和 JavaScript 分割為多個模塊文件，按需加載。避免單個文件過大。

**圖像優化**
所有圖像應進行尺寸優化（通常不超過必要的顯示尺寸的 2 倍）。優先使用現代圖像格式（WebP），並提供備用格式（JPG、PNG）。頭像等小圖像可使用 SVG 格式。

**CSS 最小化**
開發環境下 CSS 可含注釋和格式，但生產環境應使用最小化（minify）的版本以減少文件大小。

**字體加載**
使用 Google Fonts CDN 加載字體。考慮設定 font-display: swap 以優化首屏加載時間，允許系統字體先顯示，Google Fonts 加載完成後再替換。

## 📋 第十部分：設計使用案例與範例

### 10.1 標準頁面模板

**基礎模板結構**
所有頁面應包含相同的 HTML 基礎結構：DOCTYPE、meta 標籤（字符集、視口設定）、head 部分（標題、字體連結、CSS 文件）、body 部分（導航欄、主內容、頁腳）。

```
  <div class="header-item">...</div>
```

**內容容器模板**

### 10.2 色彩應用示例

**卡片設計**
卡片背景採用白色（#ffffff），邊框可選（1px #A0A0A0 或無邊框），陰影為 box-shadow: 0 4px 16px rgba(60, 60, 120, 0.08)。卡片內文字採用深灰色（#121212）。

**強調元素**
需要強調的元素（如按鈕、連結）可採用 #6366f1 或 #60a5fa 等漸變色。

**禁用狀態**
禁用的按鈕或輸入欄應採用淺灰色（#ECECEC）背景、灰色文字（#A0A0A0），opacity 可設定為 0.5，表示禁用狀態。

### 10.3 排版應用示例

**頁面標題排版**
頁面主標題採用 Inter 字體、font-size: clamp(3rem, 8vw, 10rem)、font-weight: 700 或 900、text-transform: uppercase。可選地使用粗細混合效果增加視覺層次。

**段落排版**
正文段落採用 font-family: Inter、font-size: 1.08rem、line-height: 1.7、color: #121212。段落之間應有 margin-bottom: 18px 的間距。

**代碼片段排版**
代碼應採用 JetBrains Mono 等寬字體，字體大小 0.9rem，背景色 #f0f0f0 或 #e8e8e8，左邊框 border-left: 3px solid #A0A0A0。

### 10.4 響應式設計應用示例

**三列轉單列**
三列卡片布局在 700px 以下應轉為單列堆疊。使用 @media (max-width: 700px) { flex-direction: column; } 實現。

**字體大小縮小**
在小屏幕上，H1 標題可採用 @media (max-width: 700px) { font-size: clamp(2rem, 5vw, 3rem); } 適度縮小字體。

**導航欄調整**
在超小屏幕上，導航欄可能需要特殊處理。如果 5 列網格在 320px 屏幕上過於擁擠，可考慮 @media (max-width: 480px) { font-size: 0.6rem; } 進一步縮小文字。

## 🎓 第十一部分：設計原則與哲學

### 11.1 核心設計原則

**極簡主義（Minimalism）**
設計中去除所有非必要元素，保留最核心的功能和視覺效果。淺灰色背景、細線邊框、極少裝飾性圖案都遵循這一原則。

**工業美學（Industrial Aesthetics）**
設計融合工業風格元素，如精確的比例、細線感、模塊化佈局、機械感的排版等，營造出專業而冷酷的氛圍。

**細線美學（Minimalist Lines）**
統一採用 1px 細線進行各種分割和邊框設計，強調精緻感。這與粗體字（font-weight: 900）的對比使用形成視覺張力。

**色彩克制（Color Restraint）**
主要色盤限制為灰色系（#ECECEC、#121212、#A0A0A0），輔助色採用紫藍漸變。避免過多彩色，確保整體統一和專業感。

**動畫適度（Moderate Animation）**
所有動畫採用統一的 0.4s cubic-bezier 過渡效果。動畫不應過快（容易顯得生硬）也不應過慢（容易顯得遲鈍）。避免不必要的動畫，只在增加用戶體驗時才使用。

**響應式優先（Responsive First）**
設計應首先考慮各種屏幕尺寸，確保在任何設備上都能提供良好體驗。而非先設計桌面版本，再被迫適配移動版本。

### 11.2 設計決策的理由

**灰色背景的選擇**
灰色背景相比白色更能減少眼睛疲勞，特別是對於需要長時間閱讀的用戶。同時，灰色是一個中立的色彩，不會對內容傳達產生干擾。

**深紫灰陰影的選擇**
相比傳統的黑色陰影，深紫灰陰影（rgba(60, 60, 120, X%)）與整體的冷色調配色系統更加協調。這使得陰影顯得更加柔和自然。

**細線邊框的選擇**
1px 細線邊框相比厚邊框更顯現代、精緻。細線設計傳達出高度的設計品味和對細節的關注。

**粗細字體對比**
在同一標題或設計元素中使用 font-weight: 100 和 font-weight: 900 的極端對比，創造出強烈的視覺焦點。這比單一字重的設計更加吸引眼球。

**固定導航欄的設計**
固定導航欄確保用戶無論在頁面何處都能快速返回首頁或訪問其他頁面，提高可用性。同時，固定導航欄也成為網站的視覺身份標誌。

## 🔒 第十三部分：設計維護與擴展指南

### 13.1 添加新頁面

當添加新頁面時，應遵循以下步驟：

1. 複製現有頁面的 HTML 基礎結構（DOCTYPE、head、body 等）
2. 在 head 中保持相同的 Google Fonts CDN 連結和 CSS 變量定義
3. 根據頁面用途選擇適當的頁面類型（英雄卡片型、側邊欄型等）
4. 復用現有的 CSS 類別和樣式，避免重新定義相同的樣式
5. 確保新頁面在導航欄中可訪問
6. 測試響應式設計，確保在各種屏幕尺寸上正常顯示

### 13.2 修改色彩系統

如需修改整個網站的色彩，只需修改 CSS 變量定義中的顏色值。由於設計中已使用 var() 函數引用這些變量，修改變量值將自動影響整個網站。

修改步驟：
1. 在 :root 選擇器中找到對應的 CSS 變量
2. 修改顏色值為新的十六進制代碼
3. 測試整個網站，確保新色彩與現有設計協調
4. 如修改輔助色，應考慮對漸變效果的影響

### 13.3 擴展字體系統

如需添加新字體或修改現有字體：

1. 在 head 的 link 標籤中添加新字體的 Google Fonts CDN 連結
2. 在 CSS 變量中添加新字體族變量（例如 --font-headings）
3. 在相應的 CSS 規則中使用新字體變量
4. 測試不同屏幕尺寸的文字顯示效果

### 13.4 優化性能

隨著內容增加，應考慮以下優化措施：

1. 定期檢查並移除未使用的 CSS 規則
2. 優化圖像尺寸和格式（WebP、優化壓縮）
3. 考慮使用 CSS Grid 或 Flexbox 的高級技巧進一步優化布局性能
4. 監測頁面加載時間，使用工具如 Google PageSpeed Insights
5. 對 JavaScript 代碼進行代碼分割和延遲加載優化
