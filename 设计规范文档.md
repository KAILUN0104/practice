## 🎨 色彩系統

### 主色盤（極簡風格主題色）

**背景色**：#ECECEC
這是整個網站的主要背景顏色，採用淺灰色調。這種選擇讓頁面看起來簡潔、專業，並減少眼睛疲勞。該顏色用於body元素、固定導航欄背景、以及所有需要中性背景的區域。

**文字色**：#121212
深灰色用於所有主要文字內容。這種深灰而非純黑色的選擇提供了溫和的對比，同時保持高度的可讀性。用於段落文字、導航標籤和主要內容區域。

**線條色**：#A0A0A0
中灰色用於所有分割線、邊框和結構性線條。這種顏色比背景色更深，但比文字色更淡，創造了精緻的視覺分層。用於header邊框、卡片邊框、過濾器欄的上下邊框等。

### 輔助色盤（About頁面和卡片設計）

**主要漸變色**：#6366f1 至 #60a5fa
這是一個從紫色到藍色的漸變。6366f1是紫羅蘭色，60a5fa是天藍色。這個漸變用於hero區塊的背景，為white about頁面的頂部創造視覺吸引力。

**背景漸變色**：#f8fafc 至 #e0e7ff
這是一個非常淡的漸變，從近白色（f8fafc）到淡紫色（e0e7ff）。用於整個about頁面的主背景，以及卡片容器的背景。

**卡片漸變色**：#e0e7ff 至 #f8fafc
這與背景漸變相反，創造了層次感。用於profile-card元素的背景，使卡片能夠從整個容器中突出。

**強調色**：#6366f1
這個紫色用於標題文字（about-content h3元素），為內容添加視覺焦點。

## 🔤 字體系統

### 主字體：Inter

**引入方式**：Google Fonts - Inter:wght@100;300;900
**用途**：所有主要的文本內容、標題和介紹文字
**字重選項**：
- 100（超薄）：用於hero section的裝飾性文字或次要標題
- 300（細體）：用於正文段落和一般內容，提供良好的可讀性
- 900（超粗）：用於主要標題和視覺層次的強調

**特色應用**：Inter字體具有現代、清潔的外觀，字母間距合理，適合數位螢幕顯示。粗細混合設計（100細體配合900粗體）能夠創造明顯的視覺層次，引導用戶的視線流動。

### 代碼/等寬字體：JetBrains Mono

**引入方式**：Google Fonts - JetBrains+Mono:wght@300;400
**用途**：導航欄標籤、時間顯示、代碼片段、技術相關的信息文字
**字重選項**：
- 300（細體）：用於導航項目標籤、日期標籤
- 400（常規）：用於代碼顯示、技術描述

**特色應用**：JetBrains Mono是專為開發者設計的等寬字體，具有優秀的可辨識度。將技術相關的文字使用等寬字體，能夠強化「專業技術」的視覺印象。

### About頁面專用：Montserrat

**引入方式**：Google Fonts - Montserrat:wght@400;700
**用途**：about.html頁面的所有文字內容
**字重選項**：
- 400（常規）：正文內容
- 700（粗體）：標題和強調文字

**特色應用**：Montserrat是幾何風格的無襯線字體，適合品牌設計和現代網站。在About頁面使用不同的字體系統，能夠為該頁面創造獨特的視覺身份。

## 📐 布局系統

### 固定導航栏（Header Layout）

**結構**：5列等寬網格布局
導航欄採用CSS Grid佈局，分為5個等寬的列。每列平均佔據20%的視口寬度。

**列布局順序**：
1. HOME - 首頁鏈接
2. TIME - 實時數位時鐘顯示
3. TAIPEI / TAIWAN - 地點標識
4. FOLLOW + - 社交媒體下拉菜單（Instagram）
5. CREATE BY LIN KAI-LUN - 製作者信息

**固定定位方式**：
- position: fixed
- top: 0
- left: 0
- width: 100%
- z-index: 1000

這種固定定位確保導航欄始終在螢幕頂部，不隨頁面滾動移動。

**高度**：50px
導航欄的高度設定為50像素，提供充足的點擊區域並保持簡潔外觀。

**背景色**：#ECECEC（同主背景色）
導航欄背景與頁面背景色相同，使其成為頁面的自然延伸，而不是獨立的界面元素。

**邊框**：1px solid #A0A0A0
導航欄下方有一條1像素的細線，用來清晰地分隔導航區域和內容區域。

**padding**：15px 20px
導航項目內部padding為上下15像素、左右20像素，確保文字有適當的呼吸空間。

**字體設置**：
- font-family: 'JetBrains Mono', monospace
- font-size: 0.75rem
- font-weight: 300
- text-transform: uppercase

等寬字體、超小字號、細體、大寫轉換共同營造出技術感和簡潔感。

**邊界分隔**：
導航欄的每個項目之間用border-right: 1px solid #A0A0A0分隔，最後一項無右邊框。這創造了一個方格狀的視覺結構。

**垂直對齊**：
使用display: flex, align-items: center確保文字在垂直方向居中。

### Body頁面空間預留

**padding-top: 60px**
為了不被固定的50px高度導航欄遮擋，body元素設定了60像素的上padding。這確保內容從導航欄下方開始。

**overflow-x: hidden**
防止水平滾動條出現，確保整個設計在各種螢幕寬度下都能正確顯示。

### 主容器佈局（主頁和產品頁）

**響應式padding**：4vw
使用視口寬度的相對單位（vw），確保padding在不同螢幕尺寸下都能保持適當的比例。

**垂直邊距**：8vh
section之間的margin設定為視口高度的8%，提供良好的垂直呼吸空間。

**Flexbox混合應用**：
主容器使用flex佈局，支持justify-content: space-between和align-items配置，創造靈活的內容排列。

### About頁面卡片佈局

**容器最大寬度**：900px
限制內容寬度為900像素，確保文字行長不會超出人類舒適的閱讀範圍。

**容器邊框**：18px border-radius
圓角設定為18像素，創造溫和、現代的外觀。

**容器陰影**：0 8px 32px rgba(60, 60, 120, 0.12)
輕微投影，模糊半徑32像素，顏色為深紫色帶12%透明度。這種陰影提供深度感但不過度。

**容器內padding**：40px 30px
卡片內部padding為上下40像素、左右30像素，為內容提供充足的空白。

**Flex內部佈局**：
- display: flex
- flex-wrap: wrap
- gap: 40px
- align-items: flex-start

使用Flexbox允許profile-card和about-content在足夠空間時並排顯示，空間不足時自動換行。40像素的gap提供卡片間的合理間距。

**Profile Card**：
- flex: 1 1 250px（最小寬度250像素，可伸縮）
- border-radius: 14px
- box-shadow: 0 4px 16px rgba(60, 60, 120, 0.08)
- padding: 30px 20px

Profile card內部有自己的漸變背景、圓角和陰影，創造卡片中卡片的視覺層次。

**About Content**：
- flex: 2 1 400px（優先占用更多空間，最小寬度400像素）

About content佔用profile card的兩倍空間，確保文字內容有足夠的寬度顯示。

### 頭像設計

**頭像尺寸**：100px × 100px
正方形頭像通過border-radius: 50%變成完美圓形。

**頭像邊框半徑**：50%（圓形）

**頭像陰影**：0 2px 8px rgba(60, 60, 120, 0.10)
輕微的投影使頭像看起來略微浮起。

**底部間距**：margin-bottom: 18px

**物件裁切**：object-fit: cover
確保頭像無論原始尺寸如何都能被正確顯示為完整圓形。

## ✨ 動畫與過渡

### 全局過渡設置

**過渡參數**：0.4s cubic-bezier(0.23, 1, 0.32, 1)
這個cubic-bezier曲線（0.23, 1, 0.32, 1）是一個特製的緩動函數，提供快速加速後緩速的效果。0.4秒（400毫秒）的持續時間足夠讓用戶感受到動畫但不會顯得過慢。

**應用場景**：
此過渡效果應用於所有hover狀態轉換、背景色變化、透明度變化等。

### 打字動畫效果

**觸發條件**：當滑鼠進入導航欄區域時
**效果**：導航項目文字被隱藏，然後逐字打出現。
**實現原理**：使用CSS animation結合@keyframes，或JavaScript操作DOM逐字顯示。
**持續時間**：通常設定為0.8秒至1秒。

## 📱 響應式設計

### 主要響應斷點

**最小響應寬度**：700px

當螢幕寬度達到700px以下時，觸發以下變化：

**容器調整**：
- flex-direction: column（改為垂直堆疊）
- padding: 30px 10px（減少左右padding）

**卡片調整**：
- Profile card和about-content都改為flex: 1 1 100%（占滿全寬）
- 頭像和文字堆疊排列
- 卡片內padding調整為適合小螢幕

**字體調整**：
雖然沒有明確的font-size media query，但使用clamp函數確保字體在不同尺寸間流暢伸縮。

**導航欄調整**（隱含）：
在極小螢幕上，5列導航可能變得很擁擠。建議未來的改進可以考慮在更小尺寸（如400px以下）改為漢堡菜單。

### 流體字體設計

**Clamp函數應用**：font-size: clamp(3rem, 8vw, 10rem)
最小字號3rem，最大字號10rem，實際字號根據視口寬度為8vw伸縮。

這確保標題在小螢幕上不會過大，在大螢幕上不會過小，實現完全的響應式字體。

## 🔧 技術實現細節

### CSS變數系統

**根元素變數**（:root）：
- --bg-color: #ECECEC
- --text-color: #121212
- --line-color: #A0A0A0
- --transition-smooth: 0.4s cubic-bezier(0.23, 1, 0.32, 1)
- --font-main: 'Inter', sans-serif
- --font-mono: 'JetBrains Mono', monospace

使用CSS變數系統允許全局換色和調整只需修改一個地方，提高維護性。

### Box Model設置

**通用設置**：
```
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
    cursor: default;
}
```

- margin和padding重置確保沒有默認間距干擾設計
- box-sizing: border-box使border和padding計入元素寬度
- cursor: default移除默認pointer光標，營造設計感

### Z-index層級

**固定導航欄**：z-index: 1000
確保導航欄始終在最上層。

**下拉菜單**：z-index: 1001
下拉菜單（follow-dropdown）的z-index略高於導航欄，確保下拉內容不被導航欄遮擋。

## 🎯 視覺設計原則

### 細線美學

整個設計的核心是使用細線（1px）分割和強調結構。
- 導航欄的分隔線
- 卡片的邊框
- 過濾器欄的上下邊框
都採用1像素的細線，創造精緻、高端的視覺感受。

### 粗細對比

通過Inter字體的100和900字重組合：
- 細體（100）用於次要信息、背景文字
- 粗體（900）用於主要標題、重點內容
這種粗細對比自動引導用戶視線，建立清晰的信息層次。

### 陰影層次

使用多層次的陰影創造深度：
- 卡片容器：0 8px 32px rgba(60, 60, 120, 0.12)（較深投影）
- Profile card：0 4px 16px rgba(60, 60, 120, 0.08)（中等投影）
- 頭像：0 2px 8px rgba(60, 60, 120, 0.10)（輕微投影）

所有陰影都使用同一個紫色基調，確保視覺統一。

### 色溫一致性

極簡風格頁面採用冷中性色（灰色），about頁面引入冷色系（紫藍漸變）。即使使用了彩色，也保持在冷色調，確保整體風格的一致性。

### 空白利用

大量的white space和padding使設計顯得寬敞、高級：
- 容器padding：40px 30px
- 項目間gap：40px
- 垂直邊距：8vh
這不是浪費空間，而是用空白來強調重要內容。

## 📸 圖片與媒體

### 頭像圖片

**容器**：width: 100px, height: 100px, border-radius: 50%

**圖片行為**：object-fit: cover
無論原始圖片比例如何，都會被裁切填充整個圓形容器。

**圖片質量建議**：
- 格式：PNG（支持透明度）或 WebP（更優的壓縮）
- 解析度：至少 200×200px（考慮2倍Retina屏）
- 文件大小：<50KB（確保網頁速度）

### 背景圖片

**漸變背景**：
使用linear-gradient而非圖片文件，確保快速加載和平滑效果。

**漸變角度**：90deg（左到右）或135deg（對角線）

## ♿ 無障礙考慮

### 色彩對比

**主要文字**：#121212 on #ECECEC
對比度約為13:1（遠超WCAG AAA標準的7:1）

**副文字**：#A0A0A0 on #ECECEC
對比度較低，應僅用於非必要信息。

### 焦點指示器

雖然設定了cursor: default，但應確保鍵盤導航時有明顯的焦點指示器（通常通過:focus偽類）。

### 字體大小

最小字體0.75rem（導航欄）基於context可能難以閱讀，建議future improvements確保導航欄在小螢幕上仍可讀。

如需切換主題或顏色方案，只需在此區塊修改數值，整個設計將隨之改變。
